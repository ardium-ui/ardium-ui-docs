Pipe for escaping HTML and XML strings in Angular templates, with a helper function for programmatic usage.

Why escape HTML?

Escaping HTML is essential to prevent unintended behaviors caused by special characters in strings. Characters like <, >, ", and others, if left unescaped, can be interpreted as HTML, which could lead to rendering issues or potential security risks, such as Cross-Site Scripting (XSS) attacks.

Using the escapeHTML pipe

The escapeHTML pipe offers a convenient way to sanitize HTML content within Angular templates, converting special characters to their HTML-safe equivalents.

By using this pipe, characters such as &, <, >, ", and ' are automatically converted to their HTML entities, making the content safe for display in the DOM.

The escapeHTML pipe is particularly helpful when displaying user-generated content, such as usernames, comments, or any other inputs from external sources, ensuring that these strings render as text rather than HTML. To use the escapeHTML pipe in your template, simply apply it to any string binding where HTML escaping is needed.

Note: escaping HTML is not needed when the text will be used with text interpolation.

Programmatic usage

For programmatic use outside of templates, the escapeHTML function offers the same HTML-escaping functionality, converting characters directly in code where template pipes aren't available.
