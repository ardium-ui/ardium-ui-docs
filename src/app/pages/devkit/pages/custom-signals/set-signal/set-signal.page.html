The Set Signal is a reactive signal designed to manage a set of unique values. It provides both mutating and non-mutating methods to interact with the set, along with computed properties like isEmpty, size, and asArray.

Set signal methods

The writable set signal exposes several mutating methods, such as add(), delete(), clear(), and set(), allowing you to modify the set in a reactive manner.

The add() method adds a value to the set, while delete() removes a value. The clear() method resets the set, and set() replaces the entire set with a new one.

Additionally, the set signal exposes the has() method, which returns true if the set contains the given element (non-reactive).

Computed properties

The Set Signal includes several computed properties that provide easy access to the state of the set:

  • isEmpty: A computed signal that returns true if the set is empty.
  • size: A computed signal that returns the current number of items in the set.
  • asArray: A computed signal that returns a shallow copy of the set as an array.
Read-only set signal

The asReadonly() method creates a read-only version of the set signal. This version exposes all the non-mutating methods and computed properties but disables any mutations (like add(), delete(), etc.), ensuring that the set's state can only be observed and not modified externally.
