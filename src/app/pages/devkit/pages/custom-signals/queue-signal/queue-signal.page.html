The Queue Signal is a reactive signal designed to manage a queue of items. It offers a full set of mutating and non-mutating methods, allowing developers to interact with the queue reactively, while preserving immutability for reactivity.

The signal supports methods such as enqueue(), dequeue(), and clear() to manipulate the queue, and provides computed properties like isEmpty, size, and front to observe the queue's state.

Manipulation methods

The writable queue signal exposes several methods to manipulate the queue, including enqueue(), dequeue(), clear(), and set(). These methods reactively update the signal's value.

The enqueue() method adds one or more items to the end of the queue, while dequeue() removes and returns the front item. The clear() method resets the queue to an empty state, and set() replaces the entire queue with a new array of items.

All methods are reactive, ensuring that updates are automatically reflected wherever the queue is consumed.

Computed Properties

The Queue Signal provides several computed properties to easily access information about the current state of the queue:

  • isEmpty: Returns true if the queue is empty.
  • size: Returns the current number of items in the queue.
  • front: Returns the first item in the queue, or undefined if the queue is empty.
  • asArray: Returns a shallow copy of the queue as an array.
Peek and front methods

The peek() method allows to get the front item of the queue without removing it. It works similarly to the front() computed property but is provided as a method for flexibility. The peek method also does not reactively update as the signal's value changes, making it useful when reactive changes are not desired.

Read-only queue signal

The asReadonly() method creates a read-only version of the queue signal. This version removes all mutating methods but retains access to the computed properties, allowing for safe consumption of the queue without the ability to modify it.
