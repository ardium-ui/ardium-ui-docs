The Map Signal is a reactive signal designed to work with JavaScript `Map` objects. It offers a full Map-like API, exposing methods for updating the value, allowing seamless and fully reactive manipulation of key-value pairs.

Map Signal Methods

The writable map signal includes several methods for manipulating the map, such as setKey(), delete(), clear(), and update(). Each method works reactively, creating a new Map on mutation to maintain immutability and reactiveness.

The setKey() method sets a value for a specific key, while delete() removes a key-value pair. The update() method allows for updating the map with an updater function, and clear() resets the map to an empty state.

Computed Properties

The Map Signal includes several reactive computed properties that reflect the current state of the Map:

  • isEmpty: Returns true if the map is empty.
  • size: Returns the size of the map.
  • entriesArray: Returns an array of [key, value] entries.
  • keysArray: Returns an array of keys.
  • valuesArray: Returns an array of values.

These computed properties are automatically updated whenever the map changes and can be used to track the current state of the map reactively.

Read-Only Map Signal

The asReadonly() method provides a read-only version of the map signal. This version disables all mutation methods and retains access to the non-mutating methods and computed properties, ensuring safe access to the map without the ability to modify it.
