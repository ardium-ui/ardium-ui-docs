The Stack Signal is a reactive signal designed to manage a stack of items. It provides mutating and non-mutating methods to interact with the stack, as well as computed properties for tracking its state.

Stack Signal Methods

The writable stack signal exposes several methods for manipulating the stack, including push(), pop(), clear(), and set(). Each method ensures that the stack remains immutable by replacing it with a new instance.

The push() method adds one or more items to the top of the stack, while pop() removes and returns the top item. The clear() method resets the stack, and set() replaces the entire stack with a new array.

Computed properties

The Stack Signal includes computed properties to easily observe the state of the stack:

  • isEmpty: Returns true if the stack is empty.
  • size: Returns the current number of items in the stack.
  • top: Returns the top item in the stack, or undefined if the stack is empty.
  • asArray: Returns a shallow copy of the stack as an array.
Peek and top methods

The peek() method allows to get the top item of the stack without removing it. It works similarly to the top() computed property but is provided as a method for flexibility. The peek method also does not reactively update as the signal's value changes, making it useful when reactive changes are not desired.

Read-only stack signal

The asReadonly() method returns a read-only version of the stack signal. This version retains the non-mutating methods and computed properties, while disabling all mutating operations, ensuring safe consumption of the stack's state.
