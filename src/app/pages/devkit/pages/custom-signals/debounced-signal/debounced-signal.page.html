The Debounced Signal delays updates to its value, applying them only after a specified period of inactivity. This is useful for reducing excessive signal updates in response to rapid input changes.

It ensures only the latest value is set after the debounce time elapses, making it ideal for user input or search filtering.

Debounce Mechanism

The signal takes an initial value and debounce time (ms). Updates are held and applied only if no new change occurs within that time.

The debounce interval can be checked using the read-only debounceTime property.
