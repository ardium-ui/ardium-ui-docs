Utility pipes for handling file extensions, file names, and file sizes in Angular templates.

Overview

The ArdiumFilePipesModule provides three utility pipes that facilitate common file-related operations in Angular templates. These include:

  • filename: Retrieves the base name of a file, excluding the extension.
  • fileext: Extracts the file extension, optionally with the dot.
  • filesize: Converts a file size (in bytes) into a human-readable format with customizable units.

Using the filename Pipe

The filename pipe retrieves the base name of a file without the extension, working with both file names as strings and File objects. Passing in null or undefined will result in an empty string. A file name containing only the extension, such as ".env", will result in the whole file name being returned.

This pipe is particularly helpful when displaying a list of file names without their extensions in the user interface.

Apply the filename pipe to any string or File object binding where the base name is required.

Using the fileext Pipe

The fileext pipe extracts the extension from a file name or a File object. It also has an option to include the dot in the extension. Passing in null or undefined will return null. A file name containing only the extension, such as ".env", will result in null as well.

This pipe is useful for displaying file types in applications that manage or display file lists.

Apply the fileext pipe to any string or File object binding where the file's extension is required.

Using the filesize Pipe

The filesize pipe converts a file size (in bytes) into a human-readable format. It provides customizable precision and an option to insert a space between the value and unit. Passing in null or undefined will result in an empty string.

This pipe is particularly useful when displaying file sizes for user-uploaded content or media in a clear, readable format.

Apply the filesize pipe to any numeric binding that represents a file size in bytes.
