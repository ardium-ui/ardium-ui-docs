An extension of the native HttpClient with support for default options and with extended type safety.

Creating an HTTP Service

Custom HTTP Services are created using the `createHttpService` function. It takes two arguments: a string containing the API url, and (optionally) default settings to be used for every request.

The function returns a class definition. To create an HTTP Service in a project, create a service that extends from the returned class definition.

Note: the native HttpClient still has to be provided with provideHttpClient()!

Why create a custom HTTP Service?

The newly created HTTP Service should be used exactly the same as the native HttpClient. All the same methods are available, but POST, PUT, and PATCH methods now support specifying the request body type through a type parameter.

The main reason to use a custom HTTP Service over HttpClient is the ability to specify the API url that will be prepended to every request, which is not easily possible with the native client.
