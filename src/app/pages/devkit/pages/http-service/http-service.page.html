An extension of the native HttpClient with support for default options and with extended type safety.

Usage

HttpService should be used exactly the same as the native HttpClient. All the same methods are available, but POST, PUT, and PATCH methods now support specifying the request body type through a type parameter.

The main reason to use HttpService over HttpClient is the ability to specify the API url that should be prepended to every request, which is not easily possible with the native client.
