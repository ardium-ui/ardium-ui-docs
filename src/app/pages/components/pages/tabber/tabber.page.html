Tabber organizes content into separate tabs, where only one tab can be displayed at once. Each tab's label is shown at the top and can be selected.

Every tab requires the tabId to be defined.
Labels

The tab's label can be customized using the "label" input.

If a more complex label is needed, create a template directly inside the tabber element. To customize tab labels individually, a template ref can be passed in to the label input.

Label styles

Additional tabber inputs can be used to style the tab labels:

  • color - defines the tab color.
  • stretchTabs - if set to true, the tabs will stretch to fill the entire width of the tabber.
  • uniformTabWidths - if set to true, all tabs will have the same width, adjusting to the width of the largest tab.
  • tabAlignment - allows the tabs to be aligned to the left, the center, or the right

If there are too many tabs to fit in a single row, they will wrap into the second row.

Events

The tabber component emits a (selectedTabIdChange) event when a tab is selected - the new tabId is emitted. Whenever a tab label is focused, a (focusedTabIdChange) event is emitted, containing the focused tabId.

The tabs themselves also emit events - (selectedChange) event is emitted when the tab is selected. (focus) and (blur) events are also emitted.

Providing default values

Default values of all tabber inputs can be provided using the provideTabberDefaults provider function. It takes an object of new default values, and returns a provider token.
