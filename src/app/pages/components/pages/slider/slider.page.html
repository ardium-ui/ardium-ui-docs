Slider allows users to select a value from a range by moving the slider thumb using mouse, touch, or keyboard, similar to a native HTML <input type="range">.

Using the slider

By default, the slider allows users to select a value between 0 and 100, and the value can be changed in increments of 1. These defaults can be changed using the min, max, and step inputs respectively. The initial value of the slider is always set to 0, unless it falls outside the defined range, in which case it is set to the nearest boundary value. The current value of the slider can be accessed using the value two-way binding.

Constraints

The min value must always be less than the max value, and the step value must be a positive number. If these constraints are not met, an error will be thrown in the console.

Selecting ranges

The range slider component allows users to select a range of values by moving two slider thumbs. It shares the same API as the regular slider component, but the value is a range object: { from: number, to: number }. By default, the value is the entire range from min to max.

Range selection behaviors

The [selectionBehavior] input controls how the range selection behaves when the two thumbs overlap or cross. It can be set to one of the following values:

  • 'allow' - allows the thumbs to overlap and cross each other freely. This is the default behavior.
  • 'push' - prevents the thumbs from crossing each other. When one thumb reaches the other, it pushes the other thumb along with it.
  • 'block' - prevents the thumbs from crossing each other. When one thumb reaches the other it becomes blocked and cannot move further.

By default, the range slider doesn't allow the thumbs to have the same value. To enable this behavior, set the [allowEqualValue] input to true.

Value ticks

By default, sliders do not display any value ticks. To enable value ticks, set the [showValueTicks] input to true. This will display ticks at each step interval along the slider track.

Tick labels

To further customize the value ticks, labels can be added using the [labels] input. This input accepts an array of numbers, each being a value at which a label will be displayed, eg. [0, 25, 50, 75, 100]. Labels can also be coerced from a string, eg. "0,25,50,75,100".

Providing default values

Default values of all slider and range slider inputs can be provided using the provideSliderDefaults provider function. It takes an object of new default values, and returns a provider token.
