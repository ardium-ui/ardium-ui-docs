Number input fields allow users to enter and adjust numerical values easily. They include increment and decrement buttons, validation for numeric input, and basic formatting options.

Using Number Inputs

Number inputs behave just like any other form control in Angular. You can bind the current numeric value two-way with [(value)] or use the component with reactive forms via [formControl]/formControlName. It accepts any number, or null for an empty input.

The component exposes a number of output events:

  • (input) - emitted on every keystroke or programmatic change, carries the current numeric value (or null when empty).
  • (change) - fires when the native input dispatches its change event, typically on blur or when the user presses enter.
  • (valueChange) - the same as change, used for two-way binding and consistent naming.
  • (quickChange) - fired when the increment/decrement buttons are activated. The payload contains the direction (1 or -1) and the new value.
  • (focus) and (blur) - standard focus/blur events.

The quickChange event is also emitted when the buttons are held down (repeat behaviour) and the component provides a [keepFocusOnQuickChangeButton] input to control whether the input should regain focus after a button click - otherwise the focus stays on the button.

Configuring the Number Input

A number input can be tailored to the range and precision required.

  • [min] and [max] - set the allowed range. The quick-change buttons automatically disable at the limits.
  • [stepSize] - the amount added or subtracted when a quick-change button is pressed. Must be a positive non-zero number. When [allowFloat] is false the step size must be an integer.

The component automatically adjusts user-typed values to the limits.

Floating-point options

By default the input only accepts integers. Set [allowFloat] to enable decimal values. When floats are allowed you can also configure:

  • [decimalSeparator] - character used to separate integer and fraction parts. Defaults to '.'. Must be a single character (e.g. ',' for European locales).
  • [maxDecimalPlaces] - maximum number of digits after the separator. A negative or non-integer value throws an error. The component automatically trims input to this precision.
  • [fixedDecimalPlaces] - when true the displayed value always shows exactly [maxDecimalPlaces] digits, padding with zeros if necessary. The user may freely edit the value however they want, and the zeroes will be added when the input is blurred.
Styling Number Inputs

Number input components offer styling options to match your design system. The inputs can be customized in 2 main ways: appearance and variant.

Appearances

Appearances control how a number input looks.

Appearance Description
filled A filled appearance with a background color. Good for standard form layouts.
outlined An outlined appearance with a border. Should be used when a cleaner look is desired.
outlined A transparent appearance without any border or background.
Variants

Variants control how a number input is shaped.

The variants rounded and sharp should generally not be used interchangeably in one app.

Variant Description
rounded Variant with slightly rounded corners. This is the default.
pill Variant with fully rounded ends.
sharp Variant with sharp, non-rounded corners.
Compact mode

Number inputs, similarly to other components, can be made smaller by using the [compact] property. This reduces padding and font-size for dense layouts such as tables or toolbars.

No buttons mode

If a custom increment/decrement implementation or a plain text field are needed, use the [noButtons] input. The value can still be changed by typing or pasting, and all other features like min, max, and step validation remain active.

Prefix & suffix

Arbitrary prefix and suffix content can be projected into the input using the ard-prefix-tmp and ard-suffix-tmp template directives. This is useful for currency symbols, units, icons, etc.

The templates are placed inside the input container and are not editable; they do not interfere with the numeric value.

Providing default values

Default values of all Number Input properties can be provided using the provideNumberInputDefaults provider function. It takes an object of new default values and returns a provider token.
