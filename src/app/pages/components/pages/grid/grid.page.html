The Grid component works well for a layout with a known number of columns. The columns can be configured with multiple breakpoints to specify the column span of each child.

Provide breakpoints

Breakpoints need to be provided separately from the component. They can be provided at the app level, or at the page level if different breakpoints are needed for different pages.

How it works

Grid is implemented with flexbox. Item widths are fluid, sized relative to their parent, and can change per breakpoint.

  • Every ard-grid is an item by default. Add container when it should wrap other grid items.
  • Grid item widths are based on columns and item size, so they stay proportional inside the container.
  • Responsive values can be passed either as breakpoint config objects or as shorthand breakpoint strings.
  • spacing, rowSpacing, and columnSpacing control gaps between items.
  • Grid is best for two-dimensional composition. Use Stack when the layout becomes purely one-dimensional.
Fluid grids

Start with a container, then assign each item a size relative to the grid's column count. By default the grid uses 12 columns.

Multiple breakpoints

Items can change their width at different breakpoints. Smaller breakpoint values apply upward until a larger breakpoint overrides them.

Responsive values

The container and its items can both respond to breakpoints. This is useful when the column count, spacing, and item sizes should all shift together.

Breakpoint values can be passed as objects, or as shorthand strings. In an object, the keys are breakpoint names and the values are the values that should apply at those breakpoints and up. In a string, the breakpoint name and value are separated by a colon, and multiple breakpoint values are separated by spaces. In both cases, the xs breakpoint must be defined.

As an example of the string syntax, size="xs:1 md:6" means the item should have a size of 1 at the xs breakpoint and up, but then change to a size of 6 at the md breakpoint and up. The xs can be omitted, resulting in a simpler string size="1 md:6".

When a breakpoint value is not defined for a property, the value from the last defined breakpoint applies. For example, if size is set to 1 at xs, and then set to 6 at md, the item will have a size of 1 from 0 to 767px, and a size of 6 from 768px and up.

The available breakpoints are in the table below.

Breakpoint Default device min width
xs 0rem (0px)
sm 36rem (576px)
md 48rem (768px)
lg 64rem (1024px)
xl 80rem (1280px)
2xl 96rem (1536px)

These breakpoints are customizable - the min width of each breakpoint can be changed, and new breakpoints can be added.

Auto-layout

Use size="grow" to let items share remaining space evenly, and size="auto" when one item should size itself to its content.

Spacing

Use spacing when the same gap should apply in both directions. Alternatively, use rowSpacing and columnSpacing when horizontal and vertical gaps should be controlled independently.

Nested grid

A grid container nested directly inside another grid container inherits layout properties such as columns and spacing unless overriden in the nested container.

Columns

Use columns to change the default 12-column system when a layout needs a different rhythm.

Limitations

Grid does not try to become every layout primitive. It is best at arranging columns and distributing items across a responsive row-based structure.

When a region should simply stack content in one direction, use ard-stack inside a grid item instead of trying to model vertical flow with Grid sizing alone.

Stack in practice

Stack is intentionally smaller in scope. It handles one-dimensional flow with the same responsive spacing and alignment ideas, but without Grid's column calculations.

Responsive direction

Stack direction can change per breakpoint, which makes it useful for controls that should stack on narrow screens and align horizontally on wider ones.

Providing default values

Default values of all grid properties can be provided using the provideGridDefaults provider function. It takes an object of new default values, and returns a provider token.

A similar function, provideStackDefaults, is available for stack component default values.
