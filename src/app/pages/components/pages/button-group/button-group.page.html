A layout component for grouping adjacent buttons into a single, cohesive control.
Why Use Button Groups

Button groups are useful when presenting related actions side by side. They visually connect actions and normalize size and border-radius between neighboring buttons.

Use button groups for segmented actions, toolbars, or compact action clusters where users should perceive controls as part of one set.

Basic Usage

Wrap two or more buttons with ard-button-group to create a connected group.

Nested Groups

Button groups can be nested to create separated clusters of actions. Nested groups will have a gap between them defined by the --ard-button-group-gap CSS variable. Nested groups inherit properties from their parent group, but can also override them as needed.

Add dividers between subsets of actions to improve scannability for larger groups.

Vertical Groups

Use the vertical property when actions should stack top-to-bottom instead of left-to-right.

Compact Groups

Use the compact property to make all child buttons compact.

Nested groups inherit the compact setting unless explicitly overridden.

Mixing Elements

Groups can contain both <ard-button> components and native elements using the ard-button directive (for example <button ard-button> or <a ard-button>).

For accessibility and semantics, use anchor elements for navigation.

Disabled State

Disable individual buttons as needed. A button group does not force all child buttons to be disabled.

Providing Default Values

Default values for button group properties can be configured globally using the provideButtonGroupDefaults provider function.
