The KBD (keyboard) component can be used to indicate user's keyboard input inside elements such as inputs, tables, text, and other.

Automatic KeyboardEvent translation

KBD provides a feature that automatically converts KeyboardEvent.code into a human-readable format.

This feature is case sensitive, so if you type it out by hand, remember to use PascalCase, just like KeyboardEvent does.

Full key names

By default, KBD will display a shortened name of the key. This can be changed using the full attribute. In some cases, however, there is no difference.

Dynamic key values

KBD allows to set the displayed key dynamically using the key attribute.

Keyboard shortcuts

Ardium UI exposes the ArdiumKbdShortcut component which streamlines the process of displaying keyboard shortcuts.

KBD Shortcut component can receive information about the shortcut either as the content, or using the keys attribute. Keys can be provided as a string of valid key names separated using a plus, a space, or a comma. If using the keys attribute, an array of key names can also be used. If changing the shortcut data dynamically is needed, keys attribute must be used.

KBD Shortcut allows to use all the same customization options as the standard KBD.

KBD appearances

KBD and KBD Shortcut components can be customized using the appearance attribute.

KBD directive

If the use case doesn't need the translation functionality and only the styling is needed, the ardKbd can be used. The appearance can be chosen using the ardKbdAppearance

Accessibility and Best Practises

ArdiumKbdComponent and ArdiumKbdShortcutComponent both use the semantic <kbd> element to ensure proper accessibility. There are some best practises advised:

  • The <ard-kbd> component should only be used to indicate keyboard input. Don't use it to mark up other types of content, as this can lead to confusion for users and affect accessibility.
  • Don't use the <ard-kbd> component just for styling or decoration. Only use it to markup text that represents actual keyboard input.
  • Always provide additional context when using <ard-kbd> component, whether be using standard text or by applying the aria-label or aria-labelledby attributes.
