A standard button component with flexible styling.
Styling Buttons

Button components offers a wide variety of styling options. The buttons can be customized in 3 main ways: color, appearance, and variant.

Colors

Button color is the most straightforward property. There are 7 button colors available:

Color Description
Button none The uncolorized (white or black) version of the button. Should be used for general, lower-priority actions.
Button prmiary The primary color of the button (default). Should be used for high-priority actions.
Button secondary The secondary color of the button.
Button danger This color signifies dangerous or destructive actions.
Button warn This color signifies a warning. Should be used for destructive actions of lower severity.
Button success This color signifies positive actions.
Button info This color should be used when paired with an informational message.
Appearances

Appearances control how a component looks.

Appearance Description
Button transparent Should be used for lowest priority actions, especially when paired with multiple options.
Button raised Should be used for medium-priority actions, with less visual emphasis that the raised-strong version.
Button raised-strong Should be used for high-priority actions or final actions.
Button outlined Should be used for medium-priority actions or when elevation is undesired.
Button outlined-strong Should be used for medium-priority actions that should draw higher attention before clicking. Usually for destructive actions.
Button flat Should be used for high-priority actions. Version of the raised-strong button but without elevation.
Variants

Variants control how a button is shaped.

The variants rounded and sharp should generally not be used interchangeably in one app.

Appearance Description
Button rounded Variant with slightly rounded corners.
Button pill Variant with fully rounded ends.
Button sharp Variant with sharp, non-rounded corners.
Disabled buttons

Button can be disabled using the disabled property. The property can accept a standard boolean value, or can be used without a value as a shorthand.

By default, disabled buttons have the pointer-events: none style applied. If the use case requires the button to still allow pointer events (e.g. hover), the pointerEventsWhenDisabled property can be set. This brings back standard pointer events and also applies cursor: not-allowed style. This is useful for example when trying to add a tooltip to the button.

Notice! When pointerEventsWhenDisabled is set to true, the button will be able to be clicked, and will emit click events, so make sure the event listener handles that correctly! The button won't be keyboard-focusable though.

Adding icons

Buttons support adding Material Icons to Ardium UI buttons. Simply specify the name of the icon in the icon field and it will be added to the button.

The icon position can additionally be controlled using the alignIcon property, which can be set to either 'left' or 'right'.

Buttons can also be made vertical, with the icon being on top or bottom of the button. Use the vertical property for that. Aligning the icon to the left will result in the icon being on top, aligning to the right will result in the icon being on the bottom.
