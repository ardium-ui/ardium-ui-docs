Icon Button is a type of button to be used with icons.
Icon Buttons Colors

Icon Button component color can be customized using the available color options.

Color Description
favorite none The uncolorized (white or black) version of the button. Should be used for general, lower-priority actions.
favorite primary The primary color of the button (default). Should be used for high-priority actions.
favorite secondary The secondary color of the button.
favorite danger This color signifies dangerous or destructive actions.
favorite warn This color signifies a warning. Should be used for destructive actions of lower severity.
favorite success This color signifies positive actions.
favorite info This color should be used when paired with an informational message.
Compact icon buttons

Icon Buttons, similarly to other components, can be made smaller by using the compact attribute.

Disabled icon buttons

Icon Button can be disabled using the disabled property. The property can accept a standard boolean value, or can be used without a value as a shorthand.

By default, disabled icon buttons have the pointer-events: none style applied. If the use case requires the button to still allow pointer events (e.g. hover), the pointerEventsWhenDisabled property can be set. This brings back standard pointer events and also applies cursor: not-allowed style. This is useful for example when trying to add a tooltip to the button.

Notice! When pointerEventsWhenDisabled is set to true, the icon button will be able to be clicked, and will emit click events, so make sure the event listener handles that correctly! The button won't be keyboard-focusable though.

Providing default values

Default values of all icon button properties can be provided using the provideIconButtonDefaults provider function. It takes an object of new default values, and returns a provider token.
